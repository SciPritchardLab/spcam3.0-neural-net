User's Guide to NCAR CAM2.0


1. Introduction

-- do not edit anything above!!!> CAM2.0.2 is the latest version of the NCAR Community Climate Model.   The primary changes since the last public release (CAM2.0.1) are incorporations of new land model data structures (CLM2.1) and a new land-atmosphere communication scheme that significantly improves the model performance. The primary changes of CAM2.0.1 relative to CAM2.0 were support for the Compaq-alpha cluster, Solaris, and Linux-PC Lahey F95 compilers. Note that answers for this version of the model as well as for CAM2.0.1 are only slightly different than the CAM2.0 release. The following provides a summary of changes for CAM2.0.1.

Note: This model release is still considered a "pre-release" of the code before a finalized version can be prepared. There are several aspects of the code that are missing from previous releases that will be provided in a future model release. Among these are: support of off-line input dataset interpolation tools, and support for reduced grid operation.

  • Model functionality:
    • Fix initialization problem with fractional land. Initialization uses the value for TS from the land initialization instead of value from udpate surface fluxes method.
    • Restart pointer files aren't looked for on MSS or local disk, and model will die if it expects to find them and can't.
    • Fix problems associated with using the scenario_so4 namelist option.
    • Various changes to the FV dynamical core to bring it up to date with the work at NASA/DAO.

  • Computational environment:

    • Added support for Compaq-Alpha cluster, Solaris and Linux PC (with Lahey F95 Compiler).
    • Model works with strict error checking with Lahey F95 compiler.
    • Support for interpolation tools for initial datasets has been temporarily removed, but datasets at several standard resolutions are provided.
    • Update ESMF library to ESMF_0_0_2p5, which changes the name of "conf.h" to "ESMF_conf.h", so that this file won't be confused with system files.
    • Use physics chunking (2D decomposition) when running the coupled model.

  • User Interface changes:
    • DIF4 is now a namelist parameter that is required to be set when running T42 resolution with Eulerian dynamics.
    • Restart files changed when running in coupled mode so that "b20.007" restart files could be used with the new model.
    • Don't limit the length on the CLM restart pointer file, doing so caused trouble when restarting cases with long caseid's.

  • Build and run scripts:
    • Add options to configure to set PCOLS (the max number of physics columns), and to turn Open-MP threading on or off.
